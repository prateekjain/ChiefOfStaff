## Quality Score Dimensions

Each entry is scored 0-100 across 5 dimensions (20 points each):

| Dimension | What It Measures | Scoring Guide |
|-----------|-----------------|---------------|
| **Adoption** | GitHub stars, downloads, community size | 0-5: unknown, 5-10: niche, 10-15: growing, 15-20: mainstream |
| **Recency** | Last commit, last release, active development | 0-5: abandoned (>1yr), 5-10: slow (>6mo), 10-15: active (monthly), 15-20: very active (weekly) |
| **Documentation** | Quality of docs, examples, tutorials | 0-5: none, 5-10: basic README, 10-15: good docs, 15-20: excellent with examples |
| **Relevance** | How useful for ChiefOfStaff ventures | 0-5: tangential, 5-10: nice-to-have, 10-15: useful, 15-20: highly relevant |
| **Community** | Issues responsiveness, Discord/forum, contributors | 0-5: dead, 5-10: minimal, 10-15: healthy, 15-20: thriving |
